---
title: "Internal Types"
description: "Wire types used for communication between the TypeScript layer and the native C++ turbo-module."
---

Source: `src/turbo-modules/InternalTypes.ts`

<Warning>
  These types are internal to the package and should not be used directly by application code. They exist to match the JSON serialization format used by the native bridge.
</Warning>

Internal types mirror the public API but use plain data properties instead of getter methods. This is because the native bridge serializes data as flat JSON objects.

## Comparison with public types

<Tabs>
  <Tab title="Public (getter-based)">
    ```typescript
    // Used by application code
    interface ISettingQuery {
      getKey(): string;
      getCategory(): SettingCategory;
      getMetadataTags(): ISettingMetadata[];
    }
    ```
  </Tab>
  <Tab title="Internal (property-based)">
    ```typescript
    // Used by the native bridge
    interface IInternalSettingQuery {
      key: string;
      category: SettingCategory;
      metadataTags: IInternalSettingMetadata[];
    }
    ```
  </Tab>
</Tabs>

</ResponseField>

## Concrete classes

The file also exports two concrete classes used for serializing responses back to native:

### InternalUpdateResultImpl

```typescript
export class InternalUpdateResultImpl implements IInternalUpdateResult {
  status: UpdateResultStatus;
  message: string;
  constructor(status: UpdateResultStatus, message: string);
}
```

### InternalSettingResponseImpl

```typescript
export class InternalSettingResponseImpl implements IInternalSettingResponse {
  key: string;
  currentValue: string;
  valueType: SettingValueType;
  readOnly: boolean;
  constructor(key: string, currentValue: string, valueType: SettingValueType, readOnly: boolean);
}
```
